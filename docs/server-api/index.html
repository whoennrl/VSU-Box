    API для стороннего сервера

    Когда пользователь устанавливает расширение, для него генерируется уникальный addon-токен (JWT). Этот токен — безопасный идентификатор пользователя для вашего сервера. Аддон-код никогда не видит токен напрямую — он только передаётся в Authorization заголовке при sdk.http.fetch().

    Срок жизни токена

    Addon-токен никогда не истекает сам по себе, но перевыпускается при переустановке расширения. Всегда проверяйте токен через addon_me — это защищает от отозванных токенов.

    addon_me — Получить данные пользователя

    Ваш сервер получает токен из заголовка Authorization: Bearer <token> и проверяет его.
    Метод GET
    URL https://vsu-box.whoennrl.ru/api/?action=addon_me
    Заголовок Authorization: Bearer <addon_token>

    Ответ (200 OK)

    {
  "ok": true,
  "addon_id":    42,
  "permissions": ["http", "autostart", "notifications"],
  "user": {
    "id":           123,
    "firstname":   "Иван",
    "lastname":    "Петров",
    "is_admin":    false,
    "is_developer": false
  }
}

    Note

    Поле user возвращается только если в разрешениях аддона есть user. Иначе — null.

    Пример на Node.js / Express

    app.get("/api/data", async (req, res) => {
    const token = req.headers.authorization?.replace("Bearer ", "")
    if (!token) return res.status(401).json({ error: "No token" })

    const check = await fetch("https://vsu-box.whoennrl.ru/api/?action=addon_me", {
        headers: { Authorization: `Bearer ${token}` }
    })
    const { ok, user, permissions } = await check.json()
    if (!ok) return res.status(403).json({ error: "Invalid token" })

    // user.id — уникальный идентификатор пользователя в VSU Box
    res.json({ data: `Привет, ${user?.firstname}!` })
})

    push_send_addon — Отправить push-уведомление

    Ваш сервер может отправить push-уведомление пользователю в любой момент — без участия аддона на стороне клиента. Типичный сценарий: пользователь совершил действие на вашем сайте → ваш бэкенд вызывает этот эндпоинт с сохранённым токеном.

    Требует разрешения notifications в манифесте
    Метод POST
    URL https://vsu-box.whoennrl.ru/api/?action=push_send_addon
    Заголовок Authorization: Bearer <addon_token>
    Content-Type application/json

    Тело запроса

    Поле Тип Обязательное Описание
    title string ✓ Заголовок уведомления
    body string ✓ Текст уведомления
    url string URL для открытия при нажатии (по умолчанию — главная страница приложения)
    icon string URL иконки уведомления (по умолчанию — иконка VSU Box)

    Пример запроса

    {
  "title": "Новое событие",
  "body":  "Вам пришло новое сообщение",
  "url":   "https://my-server.example.com/events/42",
  "icon":  "https://my-server.example.com/icon.png"
}

    Ответ

    { "ok": true, "sent": 2 }

    sent — количество доставленных устройств.

    Возможные ошибки

    Код Ответ Причина
    403 { "ok": false, "error": "Нет разрешения: notifications" } В манифесте нет permissions: ["notifications"]
    401 { "ok": false, "error": "Токен отозван" } Пользователь переустановил расширение

})

    Как получить токен на сервере

    Сохраните токен, который пришёл в заголовке Authorization при первом запросе к вашему серверу. Проверьте его актуальность через addon_me перед сохранением — это гарантирует, что токен не был отозван.

    sent = 0

    Если пользователь не дал браузеру разрешение на push-уведомления или не установил приложение на устройство, sent будет 0. Это нормально.
